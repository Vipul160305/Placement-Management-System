High Priority (core functionality gaps)
1. Drive Edit in TPO UI The backend supports PATCH /api/drives/:id but the DrivesPage has no edit form. TPOs can only create/delete, not fix mistakes.

2. Application Management UI There's no page for TPO/coordinators to actually view and update application statuses (shortlist, offer, reject). The backend is fully built — the frontend is missing it.

3. Resume Upload Currently a stub that returns fake success. Students have no way to attach a resume to their profile or application. Backend route needs building.

4. Notifications useNotificationSocket exists but likely does nothing real. Students should get notified when their application status changes (shortlisted/offered/rejected).

Medium Priority (UX & operations)
5. Pagination Lists are hard-capped at 200/500 docs. As data grows this becomes a problem. Add cursor or page-based pagination on drives, applications, users.

6. Student Profile Edit Students can't update their own CGPA, backlogs, branch, or section. They're stuck with whatever was imported. A self-edit profile page would fix this.

7. Drive Description Display The description field exists on the Drive model but is never shown anywhere in the UI — not in the student drive list, not in the TPO details panel.

8. Coordinator Application View Coordinators can shortlist/reject via API but there's no dedicated UI page for them to see and act on applications for their department.

Lower Priority (nice to have)
9. Admin Audit Log Viewer AuditLog model is fully populated but there's no UI to browse it. Admins are flying blind.

10. Password Reset Flow No forgot password / reset flow exists. Right now if a student forgets their password, an admin has to manually update it.

11. Drive Search/Filter for TPO The TPO drives page only has status tabs — no search by company name or title.

12. Accidental file cleanup StudentDashboard.jsx,waitForPreviousTools is sitting in the pages folder and should be deleted.
